package com.tcs.training.entity;

import jakarta.persistence.Entity;
import jakarta.persistence.GeneratedValue;
import jakarta.persistence.GenerationType;
import jakarta.persistence.Id;
import jakarta.persistence.Table;
import lombok.Data;

@Entity
@Table(name = "posts")
@Data                                // Lombok: getters, setters, toString, equals, hashCode
public class Post {

    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    private String title;
    private String description;
    private String content;
}
What @Data buys you Lombok's @Data generates all getters, setters, equals(), hashCode() and toString(). You write the four fields — Lombok writes the rest. That's ~40 lines you don't have to type or maintain.
Expected The class compiles with no errors. When you (re)start the app later, Hibernate will create a POSTS table from this class automatically.
SymptomCauseFix
@Data not resolving / "cannot find symbol getTitle()" Annotation processing not enabled in the IDE. IntelliJ: Settings → Build, Execution, Deployment → Compiler → Annotation Processors → tick "Enable annotation processing". Eclipse: install lombok.jar (run it, point it at your Eclipse) and restart.
Import errors on javax.persistence.* Spring Boot 3 uses the jakarta namespace, not javax. Import from jakarta.persistence.* as shown above.

import com.tcs.training.entity.Post;
import org.springframework.data.jpa.repository.JpaRepository;
import org.springframework.stereotype.Repository;

@Repository
public interface PostRepository extends JpaRepository<Post, Long> {
    // No methods needed — JpaRepository provides save(), findById(),
    // findAll(), deleteById() and many more out of the box.

One interface, full CRUD This single interface gives you save(), findById(), findAll(), deleteById(), count() and more — no implementation class required. Spring Data generates it at runtime.

package com.tcs.training.dto;

import lombok.Data;

@Data
public class PostDTO {

    private Long id;
    private String title;
    private String description;
    private String content;
}
PostDTO vs Post — same fields, different purpose They look almost identical, and that's fine. The Entity maps to the database table and may grow internal fields later (timestamps, author id, soft-delete flag). The DTO is the public face — it only ever exposes what the API should return. Keeping them separate means changing the table never accidentally changes your API.
LAB 4▶ Create the Custom Exception
Concept A custom exception lets us return a clean, meaningful error. It extends RuntimeException, takes a message, and carries @ResponseStatus(HttpStatus.NOT_FOUND) so Spring responds with HTTP 404 automatically when it's thrown.
Do this — step by step
  1. Right-click com.tcs.training → New → Package → com.tcs.training.exception.
  2. Right-click the new exception package → New → Class → name ResourceNotFoundException → Finish.
  3. Replace the file's contents with the code below.
📄com/tcs/training/exception/ResourceNotFoundException.javapaste the WHOLE file
package com.tcs.training.exception;

import org.springframework.http.HttpStatus;
import org.springframework.web.bind.annotation.ResponseStatus;

@ResponseStatus(HttpStatus.NOT_FOUND)        // <-- Spring returns 404 when this is thrown
public class ResourceNotFoundException extends RuntimeException {

    public ResourceNotFoundException(String message) {
        super(message);                      // the message the client will see
    }
}
What @ResponseStatus does here It tells Spring: "whenever this exception bubbles up unhandled, set the HTTP response code to 404 Not Found." No try/catch, no extra config — the annotation does the wiring.

📄com/tcs/training/service/PostServiceImpl.javafile ② — implementation
package com.tcs.training.service;

import com.tcs.training.dto.PostDTO;
import com.tcs.training.entity.Post;
import com.tcs.training.exception.ResourceNotFoundException;
import com.tcs.training.repository.PostRepository;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

import java.util.List;
import java.util.stream.Collectors;

@Service
public class PostServiceImpl implements PostService {

    @Autowired
    private PostRepository postRepository;

    // ===== Helper mappers — the core of the DTO pattern =====

    // Entity -> DTO  (going OUT to the client)
    private PostDTO mapToDTO(Post post) {
        PostDTO dto = new PostDTO();
        dto.setId(post.getId());
        dto.setTitle(post.getTitle());
        dto.setDescription(post.getDescription());
        dto.setContent(post.getContent());
        return dto;
    }

    // DTO -> Entity  (coming IN from the client)
    private Post mapToEntity(PostDTO dto) {
        Post post = new Post();
        post.setTitle(dto.getTitle());
        post.setDescription(dto.getDescription());
        post.setContent(dto.getContent());
        return post;
    }

    // ===== Business methods =====

    @Override
    public PostDTO createPost(PostDTO postDTO) {
        Post post = mapToEntity(postDTO);   // DTO -> Entity
        Post saved = postRepository.save(post);
        return mapToDTO(saved);             // Entity -> DTO
    }

    @Override
    public List<PostDTO> getAllPosts() {
        return postRepository.findAll()
                .stream()
                .map(this::mapToDTO)
                .collect(Collectors.toList());
    }

    @Override
    public PostDTO getPostById(Long id) {
        Post post = postRepository.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException(
                        "Post not found with id = " + id));
        return mapToDTO(post);
    }

    @Override
    public PostDTO updatePost(Long id, PostDTO postDTO) {
        Post post = postRepository.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException(
                        "Post not found with id = " + id));
        post.setTitle(postDTO.getTitle());
        post.setDescription(postDTO.getDescription());
        post.setContent(postDTO.getContent());
        Post updated = postRepository.save(post);
        return mapToDTO(updated);
    }

    @Override
    public void deletePost(Long id) {
        Post post = postRepository.findById(id)
                .orElseThrow(() -> new ResourceNotFoundException(
                        "Post not found with id = " + id));
        postRepository.delete(post);
    }
}
orElseThrow — read it like English findById(id) returns an Optional<Post>. orElseThrow() says: "if a Post was found, give it to me; if not, throw this exception." No null checks, no NullPointerException — and the client gets the clean 404 message you wrote.
Try it yourself Add a method long countPosts() to the interface and implement it with postRepository.count(). Wire it to a GET /api/posts/count endpoint in the next lab and watch it return the number of rows.

package com.tcs.training.controller;

import com.tcs.training.dto.PostDTO;
import com.tcs.training.service.PostService;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.List;

@RestController
@RequestMapping("/api/posts")          // base path: every method is relative to this
public class PostController {

    @Autowired
    private PostService postService;

    // CREATE -> 201 Created
    @PostMapping
    public ResponseEntity<PostDTO> createPost(@RequestBody PostDTO postDTO) {
        PostDTO created = postService.createPost(postDTO);
        return new ResponseEntity<>(created, HttpStatus.CREATED);
    }

    // READ all -> 200 OK
    @GetMapping
    public List<PostDTO> getAllPosts() {
        return postService.getAllPosts();
    }

    // READ one -> 200 OK (or 404 via the service)
    @GetMapping("/{id}")
    public ResponseEntity<PostDTO> getPostById(@PathVariable Long id) {
        return ResponseEntity.ok(postService.getPostById(id));
    }

    // UPDATE -> 200 OK (or 404 via the service)
    @PutMapping("/{id}")
    public ResponseEntity<PostDTO> updatePost(@PathVariable Long id,
                                              @RequestBody PostDTO postDTO) {
        return ResponseEntity.ok(postService.updatePost(id, postDTO));
    }

    // DELETE -> 200 OK with a confirmation message
    @DeleteMapping("/{id}")
    public ResponseEntity<String> deletePost(@PathVariable Long id) {
        postService.deletePost(id);
        return ResponseEntity.ok("Post deleted successfully with id = " + id);
    }
}
Class-level @RequestMapping Because @RequestMapping("/api/posts") sits on the class, each method only needs its suffix (/{id}) or nothing at all. The full path is assembled for you: /api/posts + /{id} = /api/posts/{id}.

BONUS▶ Spring Security — observe only (no coding)
Concept Adding Spring Security to the classpath locks down every endpoint with a default login page and a generated password — before you write any security code. This is an observe-only section: the trainer drives, you watch.
  1. Add this dependency to pom.xml:
<dependency>
    <groupId>org.springframework.boot</groupId>
    <artifactId>spring-boot-starter-security</artifactId>
</dependency>
  1. Restart the app.
  2. Navigate to http://localhost:8080/api/posts in the browser — a login page appears automatically.
  3. Check the console for the generated password line: 
    Using generated security password: 8f3c1d2e-....-a1b2c3d4e5f6
  4. Log in with username user and that generated password.
  5. Override the defaults — add these two lines to application.properties:
spring.security.user.name=admin
spring.security.user.password=admin123
  1. Restart and log in with admin / admin123 — endpoints are now accessible.
Note This is form-based security — fine for a quick lockdown. Production APIs use token-based security (JWT) with roles, which is a separate, dedicated session. Today's point is just how little it takes to switch protection on.
Tip — testing after this Once security is on, Postman requests need Basic Auth (username + password) in the Authorization tab, or they'll get 401 Unauthorized. If you want to keep testing freely, remove the dependency again after the demo.
