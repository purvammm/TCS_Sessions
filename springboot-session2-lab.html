The labs — build the API
LAB 1▶ Create the Student Bean
Concept A bean here just means a plain Java class that holds data — our Student. It has private fields, a constructor, and getters/setters. Spring's Jackson library uses the getters to turn a Student into JSON, and the setters to turn incoming JSON back into a Student.
  1. Create a package com.tcs.training.model.
  2. Create a class Student.java inside it with the code below.
package com.tcs.training.model;

public class Student {

    private int id;
    private String firstName;
    private String lastName;
    private String email;

    // No-arg constructor (Jackson needs this to build objects from JSON)
    public Student() {
    }

    // Parameterized constructor (handy for hardcoding test data)
    public Student(int id, String firstName, String lastName, String email) {
        this.id = id;
        this.firstName = firstName;
        this.lastName = lastName;
        this.email = email;
    }

    // Getters and setters
    public int getId() { return id; }
    public void setId(int id) { this.id = id; }

    public String getFirstName() { return firstName; }
    public void setFirstName(String firstName) { this.firstName = firstName; }

    public String getLastName() { return lastName; }
    public void setLastName(String lastName) { this.lastName = lastName; }

    public String getEmail() { return email; }
    public void setEmail(String email) { this.email = email; }
}
Why we write getters/setters by hand You could use Lombok's @Data to generate these automatically — but we're writing them manually today so you see exactly what they do. Jackson calls getFirstName() to produce the "firstName" JSON key. The names matter.
Expected The class compiles with no errors. There's nothing to run yet — the controller in Lab 2 will use it.
LAB 2GET▶ Controller + Your First GET Endpoint
Concept @RestController marks this class as a REST handler whose return values become JSON. @RequestMapping("/students") at the class level means every method's URL starts with /students — you don't repeat it on each method.
  1. Create a package com.tcs.training.controller.
  2. Create StudentController.java with the code below.
  3. Run the app and open http://localhost:8080/students in your browser.
  4. Open Swagger at /swagger-ui/index.html and confirm the endpoint is now listed.
package com.tcs.training.controller;

import com.tcs.training.model.Student;
import org.springframework.web.bind.annotation.GetMapping;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RestController;

@RestController
@RequestMapping("/students")   // class-level prefix: every method starts with /students
public class StudentController {

    @GetMapping   // maps GET /students
    public Student getStudent() {
        return new Student(1, "Ravi", "Sharma", "ravi@tcs.com");
    }
}
Expected response — GET /students 
{
  "id": 1,
  "firstName": "Ravi",
  "lastName": "Sharma",
  "email": "ravi@tcs.com"
}
SymptomCauseFix
Browser shows an error page / "Whitelabel Error" Forgot @RestController, so Spring doesn't register the class. Add @RestController above the class and re-run.
404 at /students Controller is outside com.tcs.training, so component scanning misses it. Move it into a sub-package of the main class's package.

Now return a List<Student>. We'll keep a shared list as a field on the controller so the later POST/PUT/DELETE labs can modify the same data.

  1. Add a field-level list of students at the top of the class (we'll reuse it everywhere).
  2. Add a GET method mapped to /students/all that returns the list.

import java.util.ArrayList;

LAB 4GET▶ GET by ID with @PathVariable
Concept @PathVariable pulls a value out of the URL path. In /students/{id} the {id} placeholder is bound to the method's int id parameter. So /students/2 calls the method with id = 2.
  1. Add a method mapped to /students/{id} that loops the list to find a match.
  2. Test http://localhost:8080/students/1 and /students/2.
import org.springframework.web.bind.annotation.PathVariable;

@GetMapping("/{id}")   // maps GET /students/{id}
public Student getStudentById(@PathVariable int id) {
    for (Student s : students) {
        if (s.getId() == id) {
            return s;
        }
    }
    return null;   // (Session 3: we'll return a proper 404 instead)
}
Expected — GET /students/2 
{ "id": 2, "firstName": "Anita", "lastName": "Verma", "email": "anita@tcs.com" }
Try it yourself Add a second endpoint @GetMapping("/name/{firstName}") that finds a student by first name. Bind it with @PathVariable String firstName and compare with equalsIgnoreCase. Expected: /students/name/Karan returns Karan's record.
LAB 5GET▶ Search with @RequestParam
Concept @RequestParam reads a query parameter — the part of the URL after the ?. In /students/search?name=Ravi, the name query value is bound to the method parameter.
  1. Add a search method mapped to /students/search with a name query param.
  2. Test http://localhost:8080/students/search?name=Ravi.
import org.springframework.web.bind.annotation.RequestParam;

@GetMapping("/search")   // maps GET /students/search?name=...
public List<Student> searchByName(@RequestParam String name) {
    List<Student> result = new ArrayList<>();
    for (Student s : students) {
        if (s.getFirstName().equalsIgnoreCase(name)) {
            result.add(s);
        }
    }
    return result;
}
Expected — GET /students/search?name=Ravi 
[ { "id": 1, "firstName": "Ravi", "lastName": "Sharma", "email": "ravi@tcs.com" } ]
PathVariable vs RequestParam — when to use which Use @PathVariable for identifying a specific resource — it's part of the resource's identity: /students/1. Use @RequestParam for filtering, searching, or optional criteria: /students/search?name=Ravi. Rule of thumb: identity in the path, filters in the query.
LAB 6POST▶ Create with @RequestBody (201 Created)
Concept @RequestBody maps the incoming JSON in the request body straight into a Java Student object — Jackson deserializes it. @ResponseStatus(HttpStatus.CREATED) makes the endpoint return 201 Created instead of the default 200.
  1. Add a POST method that takes a @RequestBody Student, adds it to the list, and is annotated with @ResponseStatus(HttpStatus.CREATED).
  2. Test in Postman: method POST, URL http://localhost:8080/students, Body → raw → JSON, and set Content-Type: application/json.
import org.springframework.http.HttpStatus;
import org.springframework.web.bind.annotation.PostMapping;
import org.springframework.web.bind.annotation.RequestBody;
import org.springframework.web.bind.annotation.ResponseStatus;

@PostMapping                              // maps POST /students
@ResponseStatus(HttpStatus.CREATED)       // returns 201 instead of 200
public String addStudent(@RequestBody Student student) {
    students.add(student);
    return "Student created with id " + student.getId();
}

Exact JSON to paste into the Postman body:

{
  "id": 4,
  "firstName": "Priya",
  "lastName": "Mehta",
  "email": "priya@tcs.com"
}
Expected Postman's response panel shows status 201 Created and the body Student created with id 4. Call GET /students/all again — Priya is now in the list.
Also try it in Swagger Expand the POST /students operation, click Try it out, paste the same JSON into the request body box, and hit Execute. Swagger shows the 201 response — no Postman needed.
LAB 7PUTDELETE▶ Update & Delete by ID

PUT — update an existing student

Concept @PutMapping("/{id}") combines both ideas: @PathVariable tells us which student to update, and @RequestBody carries the new data. 
import org.springframework.web.bind.annotation.PutMapping;

@PutMapping("/{id}")   // maps PUT /students/{id}
public Student updateStudent(@PathVariable int id, @RequestBody Student updated) {
    for (int i = 0; i < students.size(); i++) {
        if (students.get(i).getId() == id) {
            updated.setId(id);          // keep the id from the URL
            students.set(i, updated);   // replace the old record
            return updated;
        }
    }
    return null;   // (Session 3: return a proper 404)
}
Test in Postman — PUT Method PUT, URL http://localhost:8080/students/2, Body → raw → JSON: { "firstName": "Anita", "lastName": "Singh", "email": "anita.singh@tcs.com" }. The response shows the updated record; GET /students/all confirms student 2 changed.

DELETE — remove a student

import org.springframework.web.bind.annotation.DeleteMapping;

@DeleteMapping("/{id}")   // maps DELETE /students/{id}
public String deleteStudent(@PathVariable int id) {
    students.removeIf(s -> s.getId() == id);
    return "Student " + id + " deleted";
}
Test in Postman — DELETE Method DELETE, URL http://localhost:8080/students/3. Response: Student 3 deleted. Call GET /students/all — Karan is gone.
Alternative: 204 No Content Instead of returning a string, you can return void and annotate the method with @ResponseStatus(HttpStatus.NO_CONTENT) to send a clean 204 — the conventional REST response for a successful delete with no body.
Important These changes only persist while the app is running. Restart it and your edits are gone — the data lives in an in-memory list. In Session 3 we connect to a real database so changes survive.
BONUS▶ ResponseEntity — do this if you finish early
Concept ResponseEntity<T> lets you control the full HTTP response — status code, headers and body. Here we refactor "GET by ID" to return 200 OK when the student exists and a clean 404 Not Found when it doesn't. This previews Session 3's exception handling without going deep. 
import org.springframework.http.ResponseEntity;

@GetMapping("/entity/{id}")   // a ResponseEntity version of GET by id
public ResponseEntity<Student> getByIdEntity(@PathVariable int id) {
    for (Student s : students) {
        if (s.getId() == id) {
            return ResponseEntity.ok(s);          // 200 + the student
        }
    }
    return ResponseEntity.notFound().build();     // 404, no body
}
Expected /students/entity/1 → status 200 with the student JSON. /students/entity/999 → status 404 Not Found with an empty body — far cleaner than the null our earlier method returned.

The entire controller as it looks after all the labs. If you fell behind, paste this, run it, and you're caught up with the group.

package com.tcs.training.controller;

import com.tcs.training.model.Student;
import org.springframework.http.HttpStatus;
import org.springframework.http.ResponseEntity;
import org.springframework.web.bind.annotation.*;

import java.util.ArrayList;
import java.util.Arrays;
import java.util.List;

@RestController
@RequestMapping("/students")
public class StudentController {

    private List<Student> students = new ArrayList<>(Arrays.asList(
            new Student(1, "Ravi", "Sharma", "ravi@tcs.com"),
            new Student(2, "Anita", "Verma", "anita@tcs.com"),
            new Student(3, "Karan", "Patel", "karan@tcs.com")
    ));

    // Lab 2 — single hardcoded student
    @GetMapping("/first")
    public Student getStudent() {
        return new Student(1, "Ravi", "Sharma", "ravi@tcs.com");
    }

    // Lab 3 — all students
    @GetMapping("/all")
    public List<Student> getAllStudents() {
        return students;
    }

    // Lab 4 — by id
    @GetMapping("/{id}")
    public Student getStudentById(@PathVariable int id) {
        for (Student s : students) {
            if (s.getId() == id) {
                return s;
            }
        }
        return null;
    }

    // Lab 5 — search by first name
    @GetMapping("/search")
    public List<Student> searchByName(@RequestParam String name) {
        List<Student> result = new ArrayList<>();
        for (Student s : students) {
            if (s.getFirstName().equalsIgnoreCase(name)) {
                result.add(s);
            }
        }
        return result;
    }

    // Lab 6 — create
    @PostMapping
    @ResponseStatus(HttpStatus.CREATED)
    public String addStudent(@RequestBody Student student) {
        students.add(student);
        return "Student created with id " + student.getId();
    }

    // Lab 7 — update
    @PutMapping("/{id}")
    public Student updateStudent(@PathVariable int id, @RequestBody Student updated) {
        for (int i = 0; i < students.size(); i++) {
            if (students.get(i).getId() == id) {
                updated.setId(id);
                students.set(i, updated);
                return updated;
            }
        }
        return null;
    }

    // Lab 7 — delete
    @DeleteMapping("/{id}")
    public String deleteStudent(@PathVariable int id) {
        students.removeIf(s -> s.getId() == id);
        return "Student " + id + " deleted";
    }

    // Bonus — ResponseEntity
    @GetMapping("/entity/{id}")
    public ResponseEntity<Student> getByIdEntity(@PathVariable int id) {
        for (Student s : students) {
            if (s.getId() == id) {
                return ResponseEntity.ok(s);
            }
        }
        return ResponseEntity.notFound().build();
    }
}
Note on URL collisions In this combined reference the single-student demo from Lab 2 is mapped to /students/first so it doesn't clash with GET /students/{id}. When two GET mappings would match the same path, Spring needs them to be distinct — a good thing to be aware of.

Quick reference — annotation cheatsheet

AnnotationWhere it goesWhat it does
@RestControllerClassMarks a REST controller; auto-serializes return values to JSON
@RequestMappingClass or MethodDefines a URI prefix (class) or path (method)
@GetMappingMethodMaps HTTP GET requests
@PostMappingMethodMaps HTTP POST requests
@PutMappingMethodMaps HTTP PUT requests
@DeleteMappingMethodMaps HTTP DELETE requests
@PathVariableParameterBinds a URI path segment to a method parameter
@RequestParamParameterBinds a query parameter to a method parameter
@RequestBodyParameterDeserializes request-body JSON into a Java object
@ResponseStatusMethodSets the response HTTP status code
