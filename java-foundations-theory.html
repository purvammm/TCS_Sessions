Topic 1 · Java Fundamentals

What is Java?

Java is a high-level, object-oriented programming language designed to be reliable, portable, and easy to maintain. You write human-readable source code, and Java turns it into instructions a computer can run.

  • Created by Sun Microsystems — led by James Gosling, with the first public release in 1995.
  • Now stewarded by Oracle, which acquired Sun in 2010; Java continues with regular releases.
  • General-purpose — used for web backends, Android apps, enterprise systems, and big-data tools.
  • Compiled and interpreted — source compiles to bytecode, which the JVM executes on any machine.
  • Strongly typed — every value has a type the compiler checks before the program runs.
Why it matters Java's promise from day one was portability and safety: write code once, run it anywhere, with the language catching many mistakes before your program ever starts.

Topic 1 · Java Fundamentals

Key features of Java

A few core features explain why Java became so widely used. We'll meet each again in more depth later in the session.

🌍

Platform independence

Compiled Java runs on any device that has a JVM — Windows, macOS, Linux — without changing or recompiling the code.

🧱

Object orientation

Programs are organised into objects that bundle data and behaviour together, making large systems easier to model and maintain.

🧹

Automatic memory management

A garbage collector reclaims memory you no longer use, so you rarely free memory by hand and avoid whole classes of bugs.

🛡️

Robust & secure

Strong typing, exception handling, and a managed runtime catch many errors early and sandbox risky operations.

Topic 1 · Java Fundamentals

"Write once, run anywhere"

This famous phrase is the heart of Java's portability. You don't compile your code separately for every operating system — you compile it once into an intermediate form called bytecode, and any JVM can run that same bytecode.

Your source code Hello.java — written once
▼ compiled by javac into
Bytecode Hello.class — platform-independent
▼ runs on any
JVM Windows · macOS · Linux — same bytecode
The key idea The bytecode does not care what machine it runs on. The JVM for each platform knows how to turn that one bytecode into instructions for its own hardware — so one compiled program runs everywhere.

Topic 1 · Java Fundamentals

Structure of a Java program

Every Java program is built from a small set of named parts. Knowing these four lets you read any Java file.

  • Class — the basic unit of a Java program. All code lives inside a class, and the file name matches the public class name.
  • The main method — the entry point. When you run a program, the JVM calls public static void main(String[] args) first.
  • Statements — the individual instructions inside methods, each ending with a semicolon, executed top to bottom.
  • Packages — named folders that group related classes and prevent naming clashes, declared with package at the top of a file.
Putting it together A package contains classes; a class contains methods (one of them often main); a method contains statements. That nesting is the shape of every Java program you will write.

Topic 1 · Java Fundamentals

Your first Java program

Here is a complete, compilable program. It declares a class, defines the main method, and prints one line to the console.

package com.example;

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello, Java!");
    }
}
  • package com.example; places the class in a package.
  • public class HelloWorld declares the class — the file is HelloWorld.java.
  • main is where execution begins; System.out.println(...) writes a line of output.

Topic 2 · The JVM

JDK, JRE, and JVM

These three names are often confused. They are nested layers — each one contains the next.

JDK Java Development Kit — everything you need to develop: the compiler (javac), tools, and a full JRE inside it.
JRE Java Runtime Environment — everything needed to run a Java program: core libraries plus the JVM inside it.
JVM Java Virtual Machine — the engine that actually loads and executes bytecode.
Remember it as JDK = JRE + development tools.   JRE = JVM + libraries.   To write code you need the JDK; to merely run it you need the JRE.

Topic 2 · The JVM

What is bytecode?

Bytecode is the intermediate instruction format the Java compiler produces from your source code. It is stored in .class files and is not tied to any particular operating system or processor.

  • Produced by javac from your .java source files.
  • Not human source and not native machine code — a compact instruction set the JVM understands.
  • Platform-independent: the same .class file runs on any JVM, which is what makes "write once, run anywhere" possible.
  • The JVM reads bytecode and translates it into instructions for the actual hardware.
Why an intermediate step? Compiling to bytecode rather than directly to machine code lets one compiled program target every platform at once — the JVM bridges the gap to each specific machine.

Topic 2 · The JVM

From source to running program

The journey from the code you type to a running program follows a fixed pipeline, in this order:

Hello.java source you write
→
javac the compiler
→
Hello.class bytecode
→
JVM executes it
  • Write your source in a .java file.
  • Compile with javac, which checks for errors and produces a .class file of bytecode.
  • Run with java, which starts the JVM and executes the bytecode.

Topic 2 · The JVM

Classloading

Before any of your code can run, the JVM must bring your classes into memory. This job is done by the class loader.

  • The JVM loads classes into memory before they are executed — a class is read from its .class file on first use.
  • Loading happens on demand: a class is loaded the first time the program needs it, not all at once up front.
  • After loading, the bytecode is verified for safety, then linked and prepared so the JVM can execute it.
The takeaway "Loading" simply means: find the .class file, read its bytecode into memory, and make it ready to run — and it always happens before execution.

Topic 2 · The JVM

JVM runtime memory areas

While your program runs, the JVM organises memory into distinct areas, each holding a specific kind of data.

Heap Holds all objects and their instance data. Shared across the whole program and managed by the garbage collector.
Stack Holds method call frames — local variables and the call chain. Each thread has its own stack.
Method area Holds class-level data: loaded class definitions, method bytecode, and static variables.

Topic 2 · The JVM

Garbage collection

Garbage collection (GC) is the JVM's automatic memory cleanup. It is what makes Java's "automatic memory management" real.

  • The garbage collector automatically reclaims heap objects that are no longer referenced by your program.
  • An object becomes eligible for collection when nothing in the program can reach it anymore.
  • You do not free memory by hand — this removes whole categories of bugs like memory leaks and dangling pointers.
In one line If your code can no longer reach an object, the JVM is free to delete it and reuse that memory — automatically, while the program runs.

Topic 2 · The JVM

JIT compilation

The JVM doesn't just interpret bytecode line by line forever. To go faster, it uses a Just-In-Time (JIT) compiler.

  • The JIT translates bytecode into native machine code at runtime, as the program executes.
  • Its purpose is to improve execution performance — native code runs much faster than interpreted bytecode.
  • "Hot" code that runs often is compiled and optimised, so long-running Java programs speed up over time.
Interpret then compile The JVM starts by interpreting bytecode for a quick start, then JIT-compiles the frequently used parts to native code for speed — getting the best of both.

Topic 2 · The JVM

Compiling and running from the terminal

These are the two commands behind the whole pipeline — javac to compile, then java to run, in that order:

javac HelloWorld.java
java HelloWorld
  • javac HelloWorld.java compiles the source and produces HelloWorld.class.
  • java HelloWorld starts the JVM and runs the class (note: no .class extension).
  • The program's output then appears in the terminal.

Topic 3 · The Eclipse IDE

What is an IDE?

An IDE (Integrated Development Environment) is a single application that bundles all the tools you need to write, build, run, and debug software in one place — instead of juggling separate programs.

⚡

Faster writing

Code completion, instant error highlighting, and refactoring help you write correct code far faster than a plain text editor.

🔨

Build & run in place

Compile and run your program with one click, with output shown right inside the IDE — no manual terminal commands.

🐞

Integrated debugging

Pause the program, step through it line by line, and inspect variables to find bugs — something a plain editor can't do.

Topic 3 · The Eclipse IDE

Eclipse as a Java IDE

Eclipse is a free, open-source IDE and one of the most popular tools for Java development. It is the environment we will use in this course.

  • Smart editor — the code editor offers code completion, syntax colouring, and real-time error underlining as you type.
  • Build and run tooling — Eclipse compiles your code automatically and runs it with a single click, showing output in a built-in console.
  • Integrated debugger — set breakpoints, step through code, and inspect variable values to understand and fix your program.
  • Project management — it organises your source files, libraries, and settings into projects so larger codebases stay tidy.
Why Eclipse for beginners It's free, runs on every major OS, and its instant feedback (errors as you type, one-click run) shortens the learning loop for newcomers.

Topic 4 · JDK & Eclipse Setup

Step 1 — Install the JDK

Before anything else, install the JDK so you have the compiler and runtime. Follow these ordered steps:

  1. Go to a JDK provider's download page (for example, Oracle JDK or an OpenJDK build such as Adoptium Temurin).
  2. Download the installer for your operating system and choose a recent Long-Term Support (LTS) version, such as Java 17 or 21.
  3. Run the installer and accept the default options to complete the installation.
  4. Ensure the JDK's bin folder is on your system PATH so the java and javac commands work in a terminal.

Topic 4 · JDK & Eclipse Setup

Step 2 — Verify the JDK

Confirm the installation worked by asking Java for its version. Open a terminal and run:

java -version

Expected result: the console displays a JDK version string, similar to this:

java version "21.0.2" 2024-01-16 LTS
Java(TM) SE Runtime Environment (build 21.0.2+13-LTS)
Java HotSpot(TM) 64-Bit Server VM (build 21.0.2+13-LTS)
What to look for If you see a version number printed, the JDK is installed and on your PATH. If instead you see "command not found," revisit the PATH step.

Topic 4 · JDK & Eclipse Setup

Step 3 — Install & launch Eclipse

With the JDK ready, install the Eclipse IDE:

  1. Download the "Eclipse IDE for Java Developers" package from the official Eclipse website.
  2. Run the Eclipse Installer (or unzip the package) and follow the prompts to install.
  3. Launch Eclipse from the installed application or shortcut.
  4. When prompted, choose a workspace folder — this is where your projects will be stored — and click Launch.

Topic 4 · JDK & Eclipse Setup

Step 4 — Create a project & class

Now make your first project and class inside Eclipse:

  1. Choose File → New → Java Project, give it a name (e.g. HelloProject), and click Finish.
  2. Right-click the project's src folder and choose New → Class.
  3. Enter a class name (e.g. HelloWorld) and tick the option to generate the public static void main method.
  4. Click Finish — Eclipse opens the new class file ready for editing.

Topic 4 · JDK & Eclipse Setup

Step 5 — Run your class

Type a line of code into main, then run it:

public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Hello from Eclipse!");
    }
}
  1. Right-click the file and choose Run As → Java Application (or click the green Run button).
  2. Expected result: the program output appears in the Eclipse Console view at the bottom — here, Hello from Eclipse!

Topic 5 · Datatypes

Primitive vs reference types

Every Java value is either a primitive or a reference type. The difference is what the variable actually holds.

  • Primitive types hold the value directly — the eight built-in types like int and boolean.
  • Reference types hold a reference (a pointer) to an object stored on the heap.
  • Objects — instances of classes — are reference types.
  • Arrays are reference types, even an array of primitives.
  • Strings (String) are objects, and therefore reference types too.
The mental picture A primitive variable is a box containing the value itself; a reference variable is a box containing the address of an object that lives elsewhere on the heap.

Topic 5 · Datatypes

Type casting & conversion

Sometimes you need to move a value from one numeric type to another. Java has two directions:

  • Widening (implicit) conversion — a smaller type fits into a larger one automatically, e.g. int → long → double. No cast needed and no data is lost.
  • Narrowing (explicit) cast — a larger type into a smaller one must be written explicitly with a cast, e.g. (int) myDouble.
Caution Narrowing casts may lose data. Converting a double to an int drops the fractional part, and a value too big for the target type will overflow into a wrong result.

Topic 5 · Datatypes

Literals

A literal is a fixed value written directly in your code. Each category of type has its own literal form:

  • Integer literal — a whole number, e.g. 42 (add L for long: 10000000000L).
  • Floating-point literal — a number with a decimal point, e.g. 3.14 (add f for float: 3.14f).
  • Character literal — a single character in single quotes, e.g. 'A'.
  • Boolean literal — either true or false.
  • String literal — text in double quotes, e.g. "Hello".

Topic 5 · Datatypes

Datatypes in code

Declaring variables of several primitive types:

int age = 30;
double price = 19.99;
char grade = 'A';
boolean isActive = true;

A narrowing cast from double to int — note the lost fraction:

double pi = 3.14159;
int whole = (int) pi;   // explicit cast; whole becomes 3

Topic 6 · Variables

Declaration vs initialization

A variable is a named container for a value. Creating one happens in two conceptual steps:

  • Declaration introduces a name and a type, telling the compiler what kind of value the variable will hold — e.g. int count;
  • Initialization assigns the variable its first value — e.g. count = 0;
  • You can do both at once: int count = 0; declares and initializes in a single line.
Why it matters Java will not let you read a local variable before it has been initialized — the compiler stops you, preventing a common source of bugs.

Topic 6 · Variables

Naming rules & conventions

Java enforces rules for variable names, and the community follows a convention on top of them.

  • A name must start with a letter, an underscore _, or a dollar sign $.
  • After the first character it may contain letters, digits, underscores, and dollar signs.
  • A name must not be a reserved keyword (you cannot name a variable class or int).
  • Names are case-sensitive — total and Total are two different variables.
  • Convention: use camelCase — start lowercase and capitalise each later word, e.g. firstName, totalAmount.

Topic 6 · Variables

Kinds of variables

Where a variable is declared determines its kind and its scope (where it can be used).

Local Declared inside a method. Exists only while that method runs; scope is the method/block.
Instance Declared inside a class but outside any method. One copy per object; scope is the whole object.
Static Declared with static in a class. One copy shared by all objects of the class; scope is the whole class.

Topic 6 · Variables

The final keyword

Use final when a value should never change after it is first set — this declares a constant.

  • A final variable cannot be reassigned once it has been initialized.
  • Trying to assign it a second time is a compile-time error, so the rule is enforced for you.
  • Constants are conventionally named in UPPER_CASE, e.g. final double PI = 3.14159;
Why use it Marking values final makes your intent clear and protects important values (like a tax rate or a limit) from being accidentally changed elsewhere.

Topic 6 · Variables

The four kinds in code

One class showing a local, an instance, a static, and a final variable together:

public class Account {
    static int totalAccounts = 0;     // static: shared by all objects
    String owner = "Unknown";          // instance: one per object
    final double MIN_BALANCE = 100.0;  // final: constant, cannot change

    void open() {
        int startingBonus = 50;        // local: exists only in this method
        totalAccounts = totalAccounts + 1;
    }
}

Topic 7 · Conditional Statements

The if family

Conditional statements let a program choose what to do based on whether a condition is true. The if family is the foundation.

  • if — runs a block only when its condition is true.
  • if-else — runs one block when true, another when false.
  • else-if ladder — chains several conditions; the first true one runs and the rest are skipped.
  • Nested if — an if placed inside another, to test a second condition only when the first holds.

Topic 7 · Conditional Statements

The switch statement

A switch compares one value against many possible case labels — cleaner than a long else-if ladder.

  • case labels list the values to match against.
  • break ends the matched case so execution leaves the switch.
  • default runs when no case matches — like a catch-all else.
  • Fall-through — if you omit break, execution continues into the next case, which is occasionally useful but often a bug.

Topic 7 · Conditional Statements

Switch expressions (Java 14+)

Modern Java (version 14 and later) added switch expressions using an arrow form that is shorter and safer.

  • Uses the arrow -> form instead of case ... : with break.
  • A switch expression yields a value, so you can assign its result directly to a variable.
  • No fall-through, so there is no need for break — each arm is self-contained.
Statement vs expression A classic switch does something; a switch expression produces a value you can store or return.

Topic 7 · Conditional Statements

The ternary operator

The ternary conditional operator is a compact one-line choice between two values, written as condition ? a : b.

  • It evaluates to a when the condition is true.
  • It evaluates to b when the condition is false.
  • Best for short, simple choices — for example picking the larger of two numbers or a label.
int max = (a > b) ? a : b;   // a if a > b, otherwise b

Topic 7 · Conditional Statements

Building conditions: operators

Conditions are built from relational and logical operators that produce a boolean result.

  • Relational operators compare two values: == (equal), != (not equal), <, >, <=, >=.
  • Logical AND && — true only when both sides are true.
  • Logical OR || — true when at least one side is true.
  • Logical NOT ! — flips true to false and false to true.
Combine them Relational operators produce the true/false facts; logical operators glue several facts together into one condition, e.g. age >= 18 && hasTicket.

Topic 7 · Conditional Statements

Conditionals in code

An if-else if-else ladder and a classic switch:

if (score >= 90) {
    grade = 'A';
} else if (score >= 80) {
    grade = 'B';
} else {
    grade = 'C';
}

switch (day) {
    case 1:  name = "Mon"; break;
    case 2:  name = "Tue"; break;
    default: name = "Other";
}

A switch expression (Java 14+) and a ternary:

String type = switch (day) {
    case 6, 7 -> "Weekend";
    default   -> "Weekday";
};

String label = (score >= 50) ? "Pass" : "Fail";
